# Emergent Knowledge Graph

## Also Known As
Zettelkasten, Wikilink Growth, Second Brain

## Problem
Knowledge organized by a single hierarchy becomes rigid. Top-down taxonomies resist cross-domain connections.

## Context
Any system accumulating knowledge over time: personal knowledge bases, research vaults, code documentation.

## Forces
Hierarchies are easy to navigate but hard to extend. Flat collections are flexible but unsearchable. Links create structure but require effort.

## Solution
Store knowledge as atomic notes connected by bidirectional links. Do not enforce a single taxonomy. Let structure emerge from link patterns. Maps of Content provide navigation landmarks. Gap detection identifies missing connections.

## Resulting Context
Graph grows organically. Cross-domain connections appear naturally. Large graphs need good MOCs. Orphan notes accumulate without maintenance.
